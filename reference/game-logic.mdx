The `Game2048` class lives in `2048.py` and owns both the board state (a 4×4 NumPy integer array) and the Tkinter GUI. All movement operations modify `self.matrix` in-place and delegate rendering to `update_grid_cells`.

---

## Constructor

### `__init__`

Initializes a new game instance. Sets up score counters, creates the empty 4×4 board, and builds the Tkinter window and grid widgets.

```python
def __init__(self):
    self.score = 0
    self.high_score = 0
    self.matrix = np.zeros((4, 4), dtype=int)  # 4x4 grid of ints
    # ... GUI setup ...
```

**Returns:** `None`

## Board management

### `reset_game`

Resets the board to a fresh state. Zeroes the score, clears the matrix, spawns two starting tiles, and redraws the grid.

```python
def reset_game(self):
    self.score = 0
    self.score_label.config(text=f"Score: {self.score}")
    self.matrix = np.zeros((4, 4), dtype=int)
    self.add_new_tile()
    self.add_new_tile()
    self.update_grid_cells()
```

**Returns:** `None`

---

### `add_new_tile`

Spawns a new tile on a randomly selected empty cell. The new tile has a value of `2` (90% probability) or `4` (10% probability). Does nothing if the board is full.

```python
def add_new_tile(self):
    empty_cells = [(i, j) for i in range(4) for j in range(4) if self.matrix[i][j] == 0]
    if empty_cells:
        i, j = random.choice(empty_cells)
        self.matrix[i][j] = 2 if random.random() < 0.9 else 4
        self.update_grid_cells()
```

**Returns:** `None`

---

### `update_grid_cells`

Redraws every cell in the Tkinter grid to match the current `self.matrix`. Empty cells (`0`) are rendered as blank with a neutral background; all other cells display their numeric value with the appropriate tile color and text color.

```python
def update_grid_cells(self):
    for i in range(4):
        for j in range(4):
            new_number = self.matrix[i][j]
            if new_number == 0:
                self.grid_cells[i][j].configure(text="", bg='#cdc1b4')
            else:
                self.grid_cells[i][j].configure(
                    text=str(new_number),
                    bg=self.get_cell_color(new_number),
                    fg=self.get_text_color(new_number)
                )
    self.window.update_idletasks()
```

**Returns:** `None`

## Movement primitives

The four directional moves are all implemented by composing three primitive operations — `stack`, `combine`, and `stack` again — after rotating or flipping the matrix so the operation always acts left-to-right.

<Note>
**Move algorithm — all directions reduce to a left move.**

Before calling `stack`/`combine`, the matrix is transformed so the target edge faces left:

- **Left** — no transformation needed.
- **Right** — `reverse` before and after.
- **Up** — `transpose` before and after.
- **Down** — `transpose` + `reverse` before; `reverse` + `transpose` after.

After applying `stack → combine → stack`, the inverse transformation restores the matrix orientation. This means only the left-move logic needs to be correct; all other directions inherit it for free.
</Note>

### `stack`

Compacts all non-zero values in each row to the left, preserving their order. Zero cells are left at the right end of each row.

```python
def stack(self):
    new_matrix = np.zeros((4, 4), dtype=int)
    for i in range(4):
        fill_position = 0
        for j in range(4):
            if self.matrix[i][j] != 0:
                new_matrix[i][fill_position] = self.matrix[i][j]
                fill_position += 1
    self.matrix = new_matrix
```

**Returns:** `None`

---

### `combine`

Merges adjacent equal tiles in each row, scanning left to right. When two neighboring cells match, the left cell doubles and the right cell becomes `0`. The merged value is added to `self.score`; if the new score exceeds `self.high_score`, the high score is updated.

```python
def combine(self):
    for i in range(4):
        for j in range(3):
            if self.matrix[i][j] != 0 and self.matrix[i][j] == self.matrix[i][j + 1]:
                self.matrix[i][j] *= 2
                self.score += self.matrix[i][j]
                if self.score > self.high_score:
                    self.high_score = self.score
                self.matrix[i][j + 1] = 0
```

**Returns:** `None`

---

### `reverse`

Flips each row horizontally (mirrors the matrix left-to-right). Used to convert right-move into left-move.

```python
def reverse(self):
    self.matrix = np.flip(self.matrix, axis=1)
```

**Returns:** `None`

---

### `transpose`

Transposes the matrix (swaps rows and columns). Used to convert up/down moves into left-move.

```python
def transpose(self):
    self.matrix = self.matrix.T
```

**Returns:** `None`

---

### `move`

Executes a full move in the given direction. Applies the appropriate sequence of primitives, then spawns a new tile if the board changed. Calls `game_over` after every move and shows the game-over popup when no moves remain.

<ParamField path="direction" type="string" required>
  Direction to move tiles. One of `"left"`, `"right"`, `"up"`, or `"down"`.
</ParamField>

```python
def move(self, direction):
    original_matrix = self.matrix.copy()

    if direction == "left":
        self.stack(); self.combine(); self.stack()
    elif direction == "right":
        self.reverse(); self.stack(); self.combine(); self.stack(); self.reverse()
    elif direction == "up":
        self.transpose(); self.stack(); self.combine(); self.stack(); self.transpose()
    elif direction == "down":
        self.transpose(); self.reverse(); self.stack(); self.combine()
        self.stack(); self.reverse(); self.transpose()

    if not np.array_equal(original_matrix, self.matrix):
        self.add_new_tile()

    if self.game_over():
        # Shows game over popup with final score and Play Again button
        pass
```

**Returns:** `None`

## Game-over detection

### `game_over`

Returns `True` when no valid moves remain. The board is considered over when all three of the following conditions hold:

1. No empty (`0`) cells exist.
2. No two horizontally adjacent cells share the same value.
3. No two vertically adjacent cells share the same value.

```python
def game_over(self):
    if 0 in self.matrix:
        return False
    for i in range(4):
        for j in range(3):
            if self.matrix[i][j] == self.matrix[i][j + 1]:
                return False
    for i in range(3):
        for j in range(4):
            if self.matrix[i][j] == self.matrix[i + 1][j]:
                return False
    return True
```

**Returns:** `bool` — `True` if no moves remain, `False` otherwise.

## Rendering helpers

### `get_cell_color`

Returns the background hex color for a given tile value. Values above `2048` use a dark fallback color (`#3c3a32`).

<ParamField path="number" type="int" required>
  The tile value to look up.
</ParamField>

```python
def get_cell_color(self, number):
    colors = {
        2: '#eee4da', 4: '#ede0c8', 8: '#f2b179',
        16: '#f59563', 32: '#f67c5f', 64: '#f65e3b',
        128: '#edcf72', 256: '#edcc61', 512: '#edc850',
        1024: '#edc53f', 2048: '#edc22e'
    }
    return colors.get(number, '#3c3a32')
```

**Returns:** `str` — A hex color string.

#### Tile color reference

| Value | Background | Notes |
|------:|------------|-------|
| 2 | `#eee4da` | Light beige |
| 4 | `#ede0c8` | Warm beige |
| 8 | `#f2b179` | Light orange |
| 16 | `#f59563` | Orange |
| 32 | `#f67c5f` | Salmon |
| 64 | `#f65e3b` | Deep orange |
| 128 | `#edcf72` | Yellow |
| 256 | `#edcc61` | Yellow |
| 512 | `#edc850` | Gold |
| 1024 | `#edc53f` | Gold |
| 2048 | `#edc22e` | Bright gold |
| >2048 | `#3c3a32` | Dark (fallback) |

---

### `get_text_color`

Returns the foreground hex color for tile text. Tiles with a value of `8` or higher use light text (`#f9f6f2`); lower values use dark text (`#776e65`) for contrast against their light backgrounds.

<ParamField path="number" type="int" required>
  The tile value to look up.
</ParamField>

```python
def get_text_color(self, number):
    return '#f9f6f2' if number >= 8 else '#776e65'
```

**Returns:** `str` — A hex color string.

## Application entry point

### `run`

Starts the Tkinter event loop. Blocks until the window is closed.

```python
def run(self):
    self.window.mainloop()
```

**Returns:** `None` (does not return until the window closes)
